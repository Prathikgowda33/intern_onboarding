# Constraints — Level 1 (AI-Assisted Build)

The acceptance checklist. Constraints are a mix of **behavioral** (process you followed),
**artifact** (what you produced), and **AI-judged** (quality of your prompting — you ask an AI to review, since there's no human mentor). Verify each
manually and self-report in [RESULTS.md](RESULTS.md).

## How to check each constraint

1. Open the relevant file in this folder.
2. Run the **How to verify** step exactly.
3. Decide **Pass** or **Fail** using the explicit **Pass if** / **Fails if** lines.
4. Write down what you observed as evidence in [RESULTS.md](RESULTS.md).

---

## Constraints

- [ ] **C1: PROMPT_LOG.md exists with at least 3 prompts**
  - How to verify: open `PROMPT_LOG.md`.
  - Pass if: the file exists. It contains at least 3 distinct prompts you sent to the AI.
    Each prompt entry includes: what you asked, the AI's response (or summary), and what
    you did with it (accepted, modified, rejected).
  - Fails if: file doesn't exist, or fewer than 3 prompts logged.

- [ ] **C2: VERIFICATION.md exists with test cases**
  - How to verify: open `VERIFICATION.md`.
  - Pass if: the file exists. It contains at least 3 test cases, each with: input (what
    Markdown you tested), expected output (what HTML you expected), actual output (what the
    tool produced), and pass/fail result.
  - Fails if: file doesn't exist, or fewer than 3 test cases, or no expected/actual columns.

- [ ] **C3: md2html.py is a working CLI tool [ARTIFACT]**
  - How to verify: run `python3 md2html.py --help`.
  - Pass if: the tool prints help text describing its usage (input file, output flag). No errors.
  - Fails if: the tool doesn't exist, crashes on `--help`, or shows no help text.

- [ ] **C4: Tool converts Markdown to HTML [ARTIFACT]**
  - How to verify: create a test file `test.md` with at least: `# Hello`, `**bold**`, `- item`, then run `python3 md2html.py test.md -o test.html && cat test.html`.
  - Pass if: `test.html` contains `<h1>Hello</h1>`, `<strong>bold</strong>`, and `<li>item</li>`
    (or equivalent valid HTML). The output is valid HTML structure.
  - Fails if: the output is not HTML, missing required elements, or the tool crashes.

- [ ] **C5: Tool handles edge cases [ARTIFACT]**
  - How to verify: run `python3 md2html.py nonexistent.md -o out.html`. Also run it without arguments: `python3 md2html.py`.
  - Pass if: the tool gives a helpful error message for a missing input file (not a Python
    traceback). The tool shows help or an error when called without arguments.
  - Fails if: unhandled crash with Python traceback.

- [ ] **C6: At least 2 sample Markdown files with HTML output**
  - How to verify: run `ls *.md *.html` from this folder.
  - Pass if: at least 2 `.md` files exist and at least 2 `.html` files exist. The `.html`
    files were generated by your tool (they contain the HTML conversion of the `.md` files).
  - Fails if: fewer than 2 of either type, or the HTML files are not generated outputs.

- [ ] **C7: Prompts show iteration and improvement [AI-JUDGED]**
  - How to verify: paste your `PROMPT_LOG.md` into Claude/ChatGPT with this prompt:
    > "Review this prompt log from someone learning AI-assisted coding. Did they iterate
    > (refine prompts based on testing), or just repeat the same request? Did they read and
    > modify the AI's output, or blindly accept it? Score the prompting quality 1-5 and
    > explain. Be critical."
  - Pass if: the prompts show a progression — initial prompt, then refinement based on testing,
    then follow-ups for edge cases. The intern clearly read and understood the AI's responses
    before the next prompt. At least one prompt references a bug or gap found during testing.
  - Fails if: all prompts are variations of the same request (no iteration), or the intern
    didn't modify the AI's output at all (blindly accepted).

- [ ] **C8: Verification is thorough [AI-JUDGED]**
  - How to verify: paste your `VERIFICATION.md` into Claude/ChatGPT with this prompt:
    > "Review this verification doc for thoroughness. Does it cover happy path AND edge
    > cases? Did the author find and fix at least one bug? Are the expected outputs
    > specific or trivially vague? Score the verification quality 1-5 and explain."
  - Pass if: the verification covers happy path AND edge cases. At least one bug was found and
    fixed (documented in VERIFICATION.md). The expected outputs are reasonable (not trivially
    empty or "it works").
  - Fails if: verification only tests the happy path, or no bugs were found/fixed, or
    expected outputs are wrong.

8 constraints. C1-C2 are behavioral (documentation of your process). C3-C6 are artifact
(tool works). C7-C8 are AI-judged (review your prompt log and verification doc with Claude/ChatGPT). If any fail,
see [../../../resources.md](../../../resources.md).
