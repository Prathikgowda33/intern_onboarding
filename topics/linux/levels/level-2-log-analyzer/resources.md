# Resources — Linux Level 2 (Shell Log Analyzer)

Study resources **only if a constraint failed**. If you passed all constraints, the level
is cleared.

This level draws from the **shared Linux resources** at
[../../resources.md](../../resources.md). Go straight to the sections relevant to your
failed constraints:

| If you failed... | Read this section in ../../resources.md |
|------------------|-----------------------------------------|
| **C1, C2** (running the script, args, permissions) | "Shell scripting basics" |
| **C4, C5, C6** (counting IPs, status codes, hour) | "Text processing: grep, sort, uniq, awk" |
| **C8** (handling a missing file gracefully) | "Robustness: exit codes, error messages" |

If you need even more foundational context (what the terminal *is*, basic navigation),
start from the top of [../../resources.md](../../resources.md) at "From absolute zero."

## How to use these

1. Open your [RESULTS.md](RESULTS.md) and find which constraints failed.
2. Read the relevant section(s) in [../../resources.md](../../resources.md).
3. Fix your script.
4. Re-run the checks in [constraints.md](constraints.md).
5. Update [RESULTS.md](RESULTS.md) with the new evidence.

Don't read everything top to bottom — target the gap.
